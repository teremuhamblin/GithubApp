# 🗺️ Roadmap — GithubApp

Ce document présente la feuille de route officielle du projet GithubApp.  
Les versions sont organisées par objectifs, stabilité et priorités.

---

## 📌 Version 1.0 — Base fonctionnelle (terminée)
- Architecture du projet
- Handlers : issues, PR, comments, commands, labeling
- Webhooks GitHub
- Logger avancé
- Workflows GitHub Actions
- Documentation interne (patterns, support, governance, etc.)

---

## 🚀 Version 1.1 — Améliorations UX & Dev
- Ajout d’un dashboard interne (statistiques webhooks)
- Ajout d’un système de configuration dynamique
- Amélioration du système de commandes
- Ajout de tests unitaires (Jest)

---

## ⚡ Version 1.2 — Automatisation avancée
- Auto‑assignation intelligente
- Auto‑labeling basé sur IA légère (mots‑clés étendus)
- Gestion automatique des reviewers
- Système de priorisation des issues

---

## 🤖 Version 1.3 — Intégration GitHub avancée
- Support des GitHub Checks
- Commentaires enrichis (embeds, tableaux, résumés)
- Analyse des PR (taille, risques, fichiers sensibles)

---

## 🌐 Version 1.4 — GitHub Pages & Documentation
- Site GitHub Pages complet
- Documentation API
- Wiki automatisé
- Génération automatique du changelog

---

## 🧠 Version 1.5 — Intelligence & Automatisation
- Suggestions automatiques sur issues
- Résumés automatiques des PR
- Analyse de code basique (linting automatique)
- Détection de doublons

---

## 📌 Vision long terme
- Plugin VSCode
- Interface web complète
- Mode multi‑projets
- Intégration Slack / Discord

## 🗺️ Cycle prévu : **v2.0 → v3.0**

---

## 🎯 v2.0 — Intelligence contextuelle & configuration avancée
- [x] Support des règles avancées dans githubapp.yml
- [x] Moteur de règles conditionnelles
- [x] Réponses contextuelles basées sur l’historique
- [x] Refactor des handlers pour logique dynamique
- [x] Optimisation du logger

---

## 🎯 v2.5 — Patterns intelligents & auto‑actions évoluées
- [x] Patterns dynamiques avec variables
- [x] Auto‑actions avancées (assign, reviewer, close, label)
- [x] Système d’actions différées (scheduler)
- [x] Amélioration du moteur de templates
- [x] Uniformisation du style rédactionnel

---

## 🎯 v3.0 — Architecture modulaire & système de plugins
- [x] Création du système de plugins GithubApp
- [x] API interne pour plugins externes
- [x] Plugin officiel : auto‑maintenance
- [x] Event bus interne
- [x] Monitoring interne (stats, logs, usage)
- [x] Refactor du cœur du bot pour modularité
- [x] Validation avancée de la configuration

---

## 🧭 Vision long terme (post‑3.0)
- [ ] Marketplace interne de plugins GithubApp
- [ ] Interface web de configuration (dashboard)
- [ ] Analytics avancées (tendances, heatmaps, activité)
- [ ] Mode “AI assistant” pour suggestions automatiques
